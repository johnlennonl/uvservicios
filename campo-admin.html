Campo

Este módulo será el punto de revisión administrativa de las jornadas que el equipo de campo registra desde su flujo operativo. Aquí podrás recibir la notificación, revisar la jornada y decidir cuándo subirla al panel interno.

Primera etapa lista

La navegación ya quedó preparada para que Campo tenga un espacio propio dentro del panel administrativo.

El siguiente paso será conectar las jornadas guardadas en campo con un feed de revisión y aprobación.

Qué llegará aquí

Bandeja de jornadas

Mostrará cada jornada enviada desde Campo con fecha, equipo de guardia, locación, pozos incluidos y resumen operativo para revisión administrativa.

Objetivo

Evitar subir sin revisar

La idea es separar captura y aprobación. Campo registra la jornada; el panel admin la valida, la corrige si hace falta y luego la sube oficialmente.

Estado actual

Listo para la siguiente iteración

Ya existe la base del flujo de Campo y ahora este espacio queda reservado para la revisión centralizada desde administración.

Flujo previsto del módulo

Así se conectará el proceso una vez implementemos la integración completa entre captura de campo y revisión administrativa.

1. Campo guarda

El operador registra la jornada y la deja almacenada como pendiente de revisión.

2. Admin recibe aviso

La nueva sección Campo mostrará la jornada recién creada dentro del panel.

3. Admin revisa

Se valida contexto, observaciones, pozos incluidos y consistencia general del reporte.

4. Admin publica

Desde aquí se aprobará la jornada y se subirá al flujo interno definitivo del sistema.
